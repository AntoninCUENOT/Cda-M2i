Résolution de problèmes

4 bugs résolus — démarche structurée

🔴 Avis privé disparaît après navigation
Cause : GET /reviews ne retourne que les avis PUBLIC — logique correcte pour les autres utilisateurs
✅ Solution : endpoint dédié GET /animes/:id/my-review — retourne l'avis de l'utilisateur connecté quelle que soit sa visibilité
🔴 Bouton Follow toujours désactivé
Cause : selectIsFollowing lisait state.social.following au lieu de state.social.followers — inversion logique
✅ Solution : corriger le selector Redux pour pointer vers le bon tableau du store
🔴 Appartenance groupe perdue au redémarrage
Cause : Redux en mémoire — isMember non rechargé au reload de l'app
✅ Solution : checkAnimeGroup() au montage → GET /groups/anime/:id retourne l'état réel en BDD
🔴 Hot-reload mort sur Windows + Docker
Cause : Docker Desktop ne propage pas les événements inotify de Windows vers le conteneur Linux
✅ Solution : backend lancé en local (npm run dev), PostgreSQL + Redis restent dans Docker
Explication de code

Middleware d'authentification JWT

export async function authenticate(req, res, next) {
// 1. Extraire le token du header
const token = req.headers.authorization
?.replace('Bearer ', '');
if (!token)
throw new AppError(401, 'Token manquant');

// 2. Blacklist Redis (logout)
const bl = await
redis.exists(`blacklist:${token}`);
if (bl)
throw new AppError(401, 'Token révoqué');

// 3. Signature + expiration
req.user = jwt.verify(token, JWT_SECRET);
next();
}

Pourquoi ce design ?

Header Authorization

Standard REST · Mobile-friendly · Pas de cookies

Blacklist Redis

10-100x plus rapide que SQL · TTL natif auto-expire

jwt.verify()

Vérifie signature + expiration · Stateless

Composabilité

authenticate + requireRole('ADMIN') = RBAC en 1 ligne
